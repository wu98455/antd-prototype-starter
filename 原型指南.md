
### 1.2 删除旧 Git 历史并初始化（必须，AI 自动执行）

复制会带上母版的 `.git`（关联 `antd-prototype-starter` 远程仓库）。**AI 在复制完成后必须立即执行本步骤，无需用户手动删除。**

## 阶段 5：修改部署配置（5 分钟，容易漏）

GitHub Pages 个人站的 Demo 地址格式：

```
https://wu98455.github.io/{项目名}/
```

**必须**让构建时的 `base` 与仓库名一致，否则部署后白屏。

### 5.1 修改 Actions 工作流

编辑 `.github/workflows/deploy-pages.yml`：

```yaml
env:
  VITE_BASE_PATH: /{项目名}/    # 改成你的仓库名，前后都要有 /
```

示例：仓库名 `guoqi-youxue-demo` → `VITE_BASE_PATH: /guoqi-youxue-demo/`

### 5.2 确认 vite 配置

`vite.config.ts` 已通过环境变量读取，**一般不用改**：

```ts
const base = process.env.VITE_BASE_PATH || '/'
```


## 阶段 6：推送代码到 GitHub（5 分钟）

### 6.1 确保代理已开启

国内 push 需要代理（Clash 端口一般为 **7892**）：

```powershell
git config --global http.proxy http://127.0.0.1:7892
git config --global https.proxy http://127.0.0.1:7892
```

### 6.2 首次提交并推送

```powershell
cd D:\dev\projects\{项目名}
git add .
git commit -m "feat: {项目中文名} 可交互原型"
git branch -M main
git remote add origin https://github.com/wu98455/{项目名}.git
git push -u origin main
```

若 `remote origin already exists`：

```powershell
git remote set-url origin https://github.com/wu98455/{项目名}.git
git push -u origin main
```

### 6.3 确认 Actions 成功

1. 打开 `https://github.com/wu98455/{项目名}/actions`
2. 等待 **Deploy to GitHub Pages** 绿色 ✅
3. 会自动创建 `gh-pages` 分支

**常见错误：**

| 错误 | 处理 |
|------|------|
| `Connection was reset` | 开代理后再 push |
| Actions 失败 | 点进日志看具体步骤 |
| 部署后白屏 | 检查 `VITE_BASE_PATH` 是否与仓库名一致 |

**产出：** 代码在 GitHub，Actions 构建成功

## 给 Cursor 的执行摘要

当用户要求「新建并部署一个原型」时，按顺序：

1. 确认 `{项目名}`、`{项目中文名}`、页面清单
2. `xcopy` 复制模板
3. **立即** `Remove-Item -Recurse -Force .git` → `git init`（AI 自动执行，不跳过）
4. 改 `.env`、`package.json`、菜单、页面、Mock
5. `pnpm install` → `pnpm dev` 验证
6. 改 `deploy-pages.yml` 中 `VITE_BASE_PATH: /{项目名}/`
7. GitHub 新建空仓库（与项目名同）
8. `git push`（代理 7892）
9. Settings → Pages → gh-pages
10. 验证 Demo 链接 + 演示脚本

**不要：** 在母版 `antd-prototype-starter` 上直接改客户业务；不要把多个客户共用一个仓库。
